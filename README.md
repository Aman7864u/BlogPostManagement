# 📝 Blog Post Management System

A full-stack web application for managing blog posts, built with **Java Spring Boot**, **Thymeleaf**, and **MySQL**. Supports complete CRUD operations — Create, Read, Update, and Delete blog posts through a clean, responsive UI.

---

## 🚀 Features

- ✅ View all blog posts in a structured table
- ✅ Add a new blog post (title, author, content)
- ✅ Edit an existing blog post with pre-filled form
- ✅ Delete a post with a confirmation page
- ✅ Clean, responsive UI with consistent design system
- ✅ MVC architecture with full separation of concerns

## ⚙️ Getting Started

### Prerequisites

Make sure you have the following installed:

- [Java JDK 17+](https://www.oracle.com/java/technologies/javase-downloads.html)
- [MySQL 8+](https://dev.mysql.com/downloads/)
- [Maven 3.6+](https://maven.apache.org/download.cgi)
- [IntelliJ IDEA](https://www.jetbrains.com/idea/) (recommended)

---

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/blogingproject.git
cd blogingproject
```

---

### 2. Create the MySQL Database

Open **MySQL Workbench** or your terminal and run:

```sql
CREATE DATABASE blogdb;
```

---

### 3. Configure `application.properties`

Open `src/main/resources/application.properties` and update with your MySQL credentials:

```properties
spring.application.name=blogingproject

# Database Configuration
spring.datasource.url=jdbc:mysql://localhost:3306/blogdb
spring.datasource.username=root
spring.datasource.password=your_password

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQLDialect
```

> ⚠️ Replace `your_password` with your actual MySQL root password.

---

### 4. Run the Application

In IntelliJ IDEA:
- Open the project
- Click the green **▶ Run** button on `BlogingprojectApplication.java`

Or via Maven in terminal:

```bash
mvn spring-boot:run
```

---

### 5. Open in Browser

```
http://localhost:8080
```

## 🗺️ Application Routes

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/` | View all blog posts |
| GET | `/new` | Show add new post form |
| POST | `/save` | Save new post to database |
| GET | `/edit/{id}` | Show edit form for a post |
| POST | `/update/{id}` | Save updated post |
| GET | `/delete/{id}` | Show delete confirmation page |
| GET | `/delete/confirm/{id}` | Permanently delete a post |

---

## 🗄️ Database Schema

The `Post` table is auto-generated by Hibernate based on the `Post.java` entity:

```sql
CREATE TABLE post (
    id      BIGINT AUTO_INCREMENT PRIMARY KEY,
    title   VARCHAR(255) NOT NULL,
    author  VARCHAR(255) NOT NULL,
    content TEXT
);
```

---

}

## 🔄 How It Works

```
Browser Request
      │
      ▼
PostController.java        ← Handles all routes (@GetMapping, @PostMapping)
      │
      ▼
PostRepo.java              ← Spring Data JPA (findAll, save, findById, deleteById)
      │
      ▼
MySQL Database (blogdb)    ← Stores all post data
      │
      ▼
Thymeleaf Template         ← Renders dynamic HTML back to browser
```

## 🐛 Common Issues & Fixes

| Issue | Fix |
|-------|-----|
| Whitelabel Error Page | Check template file name matches `return "templateName"` in controller exactly |
| `listPosts` not showing data | Ensure `model.addAttribute("listPosts", ...)` — name must match `${listPosts}` in HTML |
| DB connection refused | Check MySQL is running and credentials in `application.properties` are correct |
| Port 8080 already in use | Add `server.port=8081` in `application.properties` |
| Table not created | Set `spring.jpa.hibernate.ddl-auto=update` in properties |
